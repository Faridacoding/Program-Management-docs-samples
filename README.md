# Program Management docs samples
This repository contains documentation samples for Product and/or Program management. It includes AI Apps for workflow automation.

**1. Documentation workflow automation** 

**Project Pulse** is an AI-Powered App which takes transcribed weekly project meeting notes files(raw transcript created by AI note taker feature available in the any video conferencing tools such as Zoom, Teams and Meet) and generates a Dashboard with project statuses and a draft newsletter for emailing the stakeholders.

This App is 'vibe coded' using prompts in Google's Gemini integrated [**AI Studio**](https://aistudio.google.com/) and uses `gemini-3-flash-preview` model.

[Demo](https://github.com/Faridacoding/Program-Management-docs-samples/blob/main/Project-Pulse/Demo.mov)\
[Project Pulse Repo](https://github.com/Faridacoding/Program-Management-docs-samples/tree/main/Project-Pulse)

**2. Sample Incident Report Triage**

This is the initial document that is created when an incident had occured. It outlines the summary of the issue and priorities assigned along with Incident commander who is in-charge of the issue.

[Sample_Incident_report_triage.pdf](https://github.com/Faridacoding/Program-Management-docs-samples/blob/main/Sample_Incident_report_triage.pdf)

**3. Sample Post Mortem Report**

This report is created after issue resolution in coordination with stakeholders and approval from all stakeholders. This report examines the RCA, Mitigation and provides Action items or next steps to avoid future occurences

[Sample_Post_Mortem_report.pdf](https://github.com/Faridacoding/Program-Management-docs-samples/blob/main/Sample_Post_Mortem_report.pdf)

**4. Sample Playbook**

This report is created by SWE or SRE's which provides instructions to the on-call engineer to help resolve the incident. 

[Sample_Playbook.pdf](https://github.com/Faridacoding/Program-Management-docs-samples/blob/main/Sample_Playbook.pdf)
